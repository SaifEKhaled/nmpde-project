Methodology III: SIMD Architecture vs. Memory Walls

1. Linear Algebra Backend & The Von Neumann Bottleneck

In standard computational physics, solving the explicit update rule requires inverting the Mass Matrix $M$. Traditionally, this is achieved by assembling local integrals into a global TrilinosWrappers::SparseMatrix utilizing the Compressed Row Storage (CRS) format. The symmetric positive-definite system is then solved using the Conjugate Gradient (CG) method.

While mathematically robust, this legacy approach triggers a massive hardware limitation: The Von Neumann Bottleneck. Modern CPU execution units idle while waiting for gigabytes of CRS arrays (values, column indices, row pointers) to travel across the limited bandwidth of the memory bus.

2. The Matrix-Free SIMD Solution

To bypass RAM latency, this engine implements a Matrix-Free Architecture. Instead of explicitly building and querying a global CRS matrix, the engine computes the action of the operator ($M\mathbf{x}$ or $K\mathbf{x}$) on-the-fly during every CG iteration. By utilizing Advanced Vector Extensions (AVX-512) for SIMD vectorization over quadrature points, the algorithm trades expensive RAM fetches for dense, cache-resident arithmetic operations.

3. Empirical Hardware Profiling

Test I: The L3 Cache Sweet Spot

Analysis: At lower resolutions, the traditional Sparse Matrix achieves 3.42 MDoPS, closely matching the Matrix-Free throughput of 3.46 MDoPS. This occurs because the entire global matrix footprint is small enough to fit inside the CPU's L3 hardware cache. Within this cache, memory latency is effectively zero, allowing CRS matrix-vector multiplication to run at peak efficiency.

Test II: The Memory Wall

Analysis: As Degrees of Freedom (DoFs) scale beyond the $2.5 \times 10^4$ L3 capacity limit, the Sparse throughput violently collapses from a peak of ~4.2 MDoPS down to 0.6 MDoPS as it hits the physical RAM bandwidth wall. Conversely, the Matrix-Free engine maintains a flat, compute-bound throughput of nearly 4.0 MDoPS regardless of grid scale.

Test III: Asymptotic RAM Footprint

Analysis: By ~60,000 DoFs, the CRS Sparse Matrix requires roughly 60 Megabytes of memory, scaling super-linearly $\mathcal{O}(N \log N)$ in 2D. The Matrix-Free method stores only the 1D solution vectors, requiring less than 1 Megabyte for the exact same resolution ($\mathcal{O}(N)$ scaling).

Test IV: Amdahl's Law (MPI Scaling)

Analysis: Strong scaling across MPI ranks reveals a massive 190% super-linear efficiency at 4 cores. This anomaly occurs when domain decomposition slices the grid perfectly into L2 cache-sized chunks. Pushing to 8 cores, efficiency drops to 85% as the MPI communication overhead of boundary synchronization begins to dominate the arithmetic compute time.
